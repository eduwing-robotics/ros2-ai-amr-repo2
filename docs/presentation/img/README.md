# 발표 이미지 업로드 폴더

여기에 아래 **정해진 파일 이름 그대로** 사진/스크린샷을 넣으면 발표 슬라이드에 자동으로 뜬다.
파일이 없으면 슬라이드에 점선 박스 + 올릴 경로가 표시되니, 그 경로에 맞춰 올리면 된다.

## 방법

1. PNG 또는 JPG 준비 (가로로 넓은 16:9 권장).
2. 파일 이름을 아래 표의 이름과 똑같이 바꾼다.
3. 이 폴더(`docs/presentation/img/`)에 넣는다.
4. `index.html`을 새로고침하면 점선 박스 자리에 사진이 뜬다.

## 필요한 이미지 목록

| 파일 이름 | 어떤 사진 | 쓰이는 덱 |
|---|---|---|
| `overview-system.png` | 전체 구성도 또는 로봇·현장 사진 (한 장으로 프로젝트를 보여줄 그림) | ① 개요 |
| `robots-two.png` | 티원(T1) + 젠지(GenG) 두 로봇 사진 | ① 개요 (선택) |
| `architecture.png` | 로봇→ROS2→Unity→DB 흐름 도식 | ② 시스템 구조 |
| `sensor-wiring.png` | 아두이노 센서 결선 사진 (PIR·레이저·사운드·온도) | ③ 로봇과 센서 |
| `control-room.png` | Unity 관제 화면 스크린샷 | ④ 관제와 데이터 |
| `live-map.png` | SLAM 지도 + 로봇 위치 마커 스크린샷 | ④ 관제와 데이터 |
| `trial-slam.png` | 깨진 SLAM 지도 / 다시 그린 지도 비교 | ⑤ 시행착오 |
| `trial-camera.png` | 카메라 끊김 진단 / 영상 뷰어 화면 | ⑤ 시행착오 |
| `scenario-flow.png` | 시나리오 흐름도 캡처 | (선택) |

> 이름·확장자는 정확히 맞춰야 한다. 대소문자도 구분. `.jpg`로 올리려면 deck 파일의 `src`도 `.jpg`로 바꾸면 된다.
