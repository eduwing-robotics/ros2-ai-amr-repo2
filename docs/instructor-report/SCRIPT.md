## 📑 슬라이드 2 — 미션 한 그림 (0:50 ~ 2:00)

이 프로젝트의 한 줄 요약입니다.
**"한 대가 발견하면, 다른 한 대가 출동한다."**

저희는 터틀봇 두 대를 **역할**로 나눠서 운용합니다.

- **1호기는 감지 담당**입니다. 정해진 경로를 순찰하면서 센서로 이상 신호를 잡습니다.
- **2호기는 출동 담당**입니다. 1호기가 보낸 좌표로 자동 길찾기 이동해서 현장에 갑니다.
- 그리고 그 모든 결과는 **유니티 관제 화면에 실시간으로 표시**되고, **데이터베이스에 자동 기록**됩니다.

쉽게 말씀드리면, **현실 로봇 두 대와 가상 관제 화면이 한 세트로 동작**하는, 가상과 실물이 연동되는 경비 시스템입니다.

## 📑 슬라이드 3 — 시나리오 4종 (2:00 ~ 3:30)

저희가 발표 때 시연할 시나리오는 네 가지입니다. 센서 보유 상태도 같이 말씀드리겠습니다.

**첫째, 야간 모드.** 조도 센서로 실내가 어두워지면 경비 모드로 **자동 전환**됩니다. 사람이 스위치를 누르지 않아도 됩니다. 단, 조도 센서 자체는 아직 키트에 포함되지 않아서 **이번 주 안에 발주 예정**입니다.

**둘째, 침입 감지.** 인체 감지 센서, 흔히 **PIR**이라고 부르는 부품을 사용합니다. **반경 약 5미터 안에서** 사람의 움직임이 잡히면 관제 화면에 위치가 표시됩니다. 이 부품은 **이미 보유 중**이라 바로 테스트할 수 있습니다.

**셋째, 이상 소음.** 소음 센서로 기준치를 넘는 소리가 나면 **2호기가 자동으로 출동**합니다. 이 센서도 키트에 없어서 **PIR과 함께 발주 예정**입니다.

**넷째, 화재 의심.** 실제 불꽃 센서를 쓰면 **불꽃을 직접 만들어야 하는 화재 위험**이 있어서 실물 테스트는 하지 않습니다. 대신 **유니티 관제 화면에 가상 화재 지점을 띄워두고**, 로봇이 그 지점을 **자동으로 우회 주행**하는 것까지 시연하겠습니다. 발표 때도 이 부분을 분명하게 말씀드릴 예정입니다.

요약하면, **PIR 한 종류는 바로 가능**하고, **조도와 소음은 발주 예정**, **화재는 유니티 시뮬레이션으로 대체**합니다.

**4번, 아두이노와 센서 4종.** 본체 상판의 빈 공간에 양면테이프로 부착합니다. **별도 적층 없이** USB 케이블로 라즈베리파이에 데이터를 보내는 단순한 구조입니다.

## 🙋 예상 질문 + 1줄 답변

| 질문 | 답변 핵심 |
|---|---|
| 왜 터틀봇 두 대인가요? | 한 대로는 "감지하면서 동시에 출동"이 안 됩니다. **역할을 분리**해야 디지털 트윈 가치가 살아납니다. |
| 디지털 트윈이 정확히 뭔가요? | **실물 로봇의 상태가 가상 관제 화면에 그대로 비치는 것**입니다. 라이다·좌표·카메라가 실시간으로 동기화됩니다. |
| 아두이노를 굳이 추가하는 이유는? | 터틀봇 기본 보드에는 **저희 시나리오용 센서 입력 핀이 부족**합니다. 별도 보드로 분리해서 안정성을 확보합니다. |
| AI는 무엇을 분류하나요? | 오탐과 실탐을 가려주는 보조 역할입니다. **모든 감지가 진짜 위협은 아니니까** 기준치와 간단한 분류로 1차로 거릅니다. |
| 7주 안에 가능한가요? | 축소안을 준비해 두었습니다. 일정 어려우면 **2대 실기 → 1대 실기 + 1대 시뮬레이션**으로 줄입니다. |
| 화재 시나리오는 어떻게 시연하나요? | 실물 불꽃 센서는 **화재 위험으로 제외**했습니다. **유니티 화면에 가상 화재 지점**을 만들어두고, **로봇이 그 지점을 우회 주행**하는 것을 시연합니다. |
| PIR 감지 거리는 얼마인가요? | 일반 모듈 기준 5~7미터인데, 실내에서 **보수적으로 약 5미터**로 잡고 테스트합니다. |
| 센서 4종은 모두 보유 중인가요? | **PIR은 보유 중**이라 바로 가능. **조도·소음은 이번 주 발주 예정**. **불꽃은 안전상 가상 시뮬레이션**으로 대체합니다. |
