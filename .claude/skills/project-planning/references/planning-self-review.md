# Planning Self-Review

구현 시작 전 아래를 확인한다.

## Scope

- MVP와 later backlog가 섞여 있지 않은가
- 이번 phase에서 하지 않을 것이 `Non-Goals`에 적혀 있는가

## Contracts

- entity, route, schema, env var 이름이 추측이 아닌가
- 애매한 임시 이름을 `banned names`에 적었는가

## Files

- 새로 만들 파일과 수정할 파일이 분리되어 있는가
- 하위 경계 폴더에서 로컬 `AGENTS.md` 검토가 필요한가

## Skill Routing

- planning, design review, implementation, contract safety, doc sync, retro 흐름이 보이는가
- 지금 단계에 필요 없는 스킬까지 과하게 적지 않았는가

## Phase Health

- 각 phase에 `Verification`, `Doc Sync`, `Exit Criteria`, `Decision Gates`가 모두 있는가
- phase가 너무 많아서 실행보다 관리가 무거워지지 않는가

## Handoff

- 다음 구현자가 어디부터 읽고 어떤 명령을 먼저 실행해야 하는지 보이는가
- blocker와 open question이 분리되어 있는가
