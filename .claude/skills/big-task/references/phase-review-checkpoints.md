# Phase Review Checkpoints

## M1 Scope Review

- 범위가 처음 의도와 맞는가
- 마일스톤이 과도하게 크지 않은가
- 로컬 `AGENTS.md`가 필요한 새 경계 폴더가 생겼는가

## M2 Implementation Review

- 구현이 기존 계약을 어기지 않는가
- 파일 길이가 300줄 근처로 커졌는가
- 재사용 가능한 공통 로직이 생겼는가

## M3 Validation Review

- validation ladder 중 최소 1단계 이상 수행했는가
- 검증 실패를 문서화했는가
- 추가 smoke test가 필요한가

## M4 Doc Sync Review

- `PROJECT-STATUS.md`를 갱신했는가
- 구조 결정이 바뀌면 `DECISION-LOG.md`에 남겼는가
- 다음 액션과 남은 리스크를 적었는가
